# Upgrade prompt on credit exhaustion

**Position**: Follow-up to Autumn billing PR (#1555)
**Estimated effort**: ~30 min

## What

When the AI chat returns 402, the error banner should show an upgrade prompt instead of a generic error message. The state layer is already wired—`ConversationHandle.isCreditsExhausted` and `aiChatState.billingUrl` exist. This adds the UI.

## Where

`ChatErrorBanner.svelte` already renders in `AiChat.svelte` when there's an error. Rather than creating a separate component, the banner should branch on whether the error is a credit exhaustion.

`AiChat.svelte` passes `active.error` and callbacks. It should also pass `active.isCreditsExhausted` and `aiChatState.billingUrl`.

## How

The change is two files:

**`AiChat.svelte`** — pass two new props to ChatErrorBanner:

```svelte
<ChatErrorBanner
    error={active.error}
    dismissedError={active.dismissedError}
    isCreditsExhausted={active.isCreditsExhausted}
    billingUrl={aiChatState.billingUrl}
    onRetry={() => { ... }}
    onDismiss={() => { ... }}
/>
```

**`ChatErrorBanner.svelte`** — branch on `isCreditsExhausted`:

```svelte
{#if visible}
    {#if isCreditsExhausted}
        <!-- Upgrade prompt: amber/warning tone, not red/destructive -->
        <div class="flex items-center justify-between gap-2 border-t border-amber-500/20 bg-amber-500/10 px-3 py-2 text-xs text-amber-200">
            <span>You've used all your credits for this cycle.</span>
            <div class="flex shrink-0 items-center gap-1">
                <a href={billingUrl} target="_blank">
                    <Button variant="outline" size="xs">Upgrade plan</Button>
                </a>
                <Button variant="ghost" size="icon-xs" onclick={onDismiss}>
                    <XIcon class="size-3" />
                </Button>
            </div>
        </div>
    {:else}
        <!-- Existing generic error banner -->
        ...
    {/if}
{/if}
```

Key design decisions:

- **Amber, not red** — credit exhaustion isn't an error in the same way a network failure is. It's a billing state. Amber communicates "action needed" without the alarm of destructive red.
- **"Upgrade plan" link, not retry** — retrying won't help. The action is upgrading. Opens the billing page in a new tab.
- **Dismiss still works** — user can dismiss the banner if they don't want to upgrade right now.
- **Copy: "You've used all your credits for this cycle."** — factual, no drama, per writing-voice skill.
- **No "Buy 500 credits" button** — keep it simple. One CTA. The billing page has all the options.
